# Product Docs

This folder captures product intent, UX behavior, domain concepts, and key decisions.

## Purpose
These docs help keep implementation aligned with the product direction.

## Rules
- Runtime truth lives in code, not in docs.
- Docs should explain why and how the product should behave.
- When a meaningful product/UX decision is made, add or update a file in `docs/decisions/`.
- Prefer small, focused updates over long speculative writeups.
